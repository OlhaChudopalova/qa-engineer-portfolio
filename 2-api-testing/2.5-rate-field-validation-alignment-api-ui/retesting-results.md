# Retesting Results

## Frontend Display

- Значення з форматом `1.0` / `1.00` після збереження відображаються як ціле число `1`.
- Поведінка UI синхронізована з бізнес-логікою та не створює плутанини для користувача.

## Rate Field Constraints

- Мінімум: `1`
- Максимум: `100000`
- Тип: integer (ціле число)
- Дробові значення з точкою, що дорівнюють цілому числу, допускаються:
 `1.0`, `1.00` → 201
- Інші дробові значення не підтримуються: `1.1` → 422
- Точка використовується як роздільник для дробів (якщо значення еквівалентне цілому числу) → 201
- Кома як роздільник не приймається:  `1,00` → 422
- Граничні значення перевірені методом 3-point Boundary Value Analysis (BVA)

## Conclusion

- API коректно обробляє граничні значення, тип даних та формат поля rate.
- Валідація відповідає бізнес-логіці.
- Поведінка UI та API синхронізована.

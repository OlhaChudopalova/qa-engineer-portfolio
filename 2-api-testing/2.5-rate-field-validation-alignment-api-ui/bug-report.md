# 🐞 Bug Report: Rate Field Validation Inconsistency Across API and UI

## Priority

Medium

## Environment

- API: http://api.stafferia.com
- Postman: v11.68.4
- Browser: Google Chrome 141
- OS: Windows 11 Home

## Endpoints

- POST `/api/v1/executor/post/profile-data/`
- GET  `/api/v1/executor/get/profile-data/`


## Precondition

- Користувач існує в системі.
- Отримано валідний access token для виконання авторизованих API-запитів.
- Користувач авторизований для перевірки відображення даних у UI на сторінці Profile.

---

## Description

Виявлено невідповідність правил валідації поля rate між Backend та Frontend, що призводить до різної поведінки системи залежно від каналу введення. 

Backend приймає значення, які:

- суперечать бізнес-логіці (наприклад, 0);
- не допускаються в UI (decimal-значення, перевищення максимальної кількості цифр).
  
Додатково встановлено, що:  

- integer-значення відображаються в UI у форматі decimal (1 → 1.00);
- GET-запит повертає значення у decimal-форматі (1.00), що порушує консистентність між правилами введення та відображення даних.

---

## Steps to Reproduce

1. Надіслати POST-запити з наступними значеннями поля `rate`:

- `0`
- `0.0`
- `1.65`
- `99999999`
- `1`

2. Перевірити відповідь Backend.
3. Перевірити відображення значення у UI.
4. Виконати GET-запит профілю.
5. Спробувати ввести decimal значення через UI (наприклад, `1.65`).

---

## Expected Result

1. Значення `0`, `0.0`, `1.65`, `99999999` (8 цифр) мають відхилятися Backend.
2. UI має відображати значення `1` без decimal-форматування.
3. GET-запит повинен повертати integer-формат (`"1"`, а не `"1.00"`).
4. Decimal-значення не повинні допускатися через UI.

---

## Actual Result

1. POST-запити успішно обробляються Backend.
2. В UI значення `1` відображається як decimal `1.00`.
3. GET-запит повертає значення у форматі  decimal `"1.00"`.
4. Неможливо ввести значення в форматі decimal через UI.

## Impact

- Неконсистентна поведінка залежно від каналу введення (API vs UI)
- Порушення бізнес-логіки
- Плутанина для користувача
- Потенційні проблеми з цілісністю даних

## Recommendation

Синхронізувати правила валідації між API та UI:

#### Backend

- Встановити максимальну кількість цифр.
- Заборонити значення 0.
- Заборонити decimal через API;
- Integer-формат у GET-відповіді.

#### Frontend

- Відображати всі значення поля rate як ціле число в UI незалежно від каналу введення.
